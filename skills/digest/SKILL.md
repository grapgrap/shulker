# Digest

현재 맥락의 document를 사용자의 목적에 맞게 종합, 정제하여 산출한다.

## 흐름

### 1. 입력 파악

사용자의 목적을 파악한다.

- 주간 보고, 동료 평가, 패턴 발견, 정리 등 어떤 목적이든 자유 형식으로 받는다
- 목적이 모호하면 무엇을 산출하려는지 물어본다

### 2. 맥락 확보

현재 맥락에 recall로 로드된 document가 있는지 확인한다.

- 있으면 그대로 사용한다
- 없으면 목적에 맞는 조건으로 recall을 직접 수행한다

### 3. 증류

로드된 document들을 리뷰하며 핵심을 드러낸다.

- 각 document에서 목적에 관련된 핵심만 추출한다
- 노이즈를 제거하고 본질을 남긴다
- 기존 document의 내용을 압축하거나 갱신할 수 있다

### 4. 종합

증류된 핵심들을 목적에 맞게 종합하여 산출물을 생성한다.

- 산출물의 형태는 목적에 따라 자유롭게 결정한다
- 사용자의 피드백을 통해 반복 정제한다

### 5. 정제

종합 과정에서 발견된 document 간 빠진 관계를 보완한다.

- 기존 document에 빠진 위키링크를 추가한다
- 필요한 stub document를 생성한다
- 문서 생성/수정 시 작성 규칙을 따른다

### 6. 저장

산출물을 document로 저장할지 판단한다.

- 사용자가 저장을 요청하면 저장한다
- 저장 시 작성 규칙을 따른다

### 7. 동기화

```bash
aeira sync -s ~/.claude/shulker/
```

## 작성 규칙

document를 생성하거나 수정할 때 따르는 규칙이다.

### 파일 이름

- 사람: `김철수.md`
- 날짜: `2026-04-17.md`
- 그 외: 최대한 짧은 단어로 구성한다
  - 날짜가 중요한 맥락: 날짜를 앞에 둔다. `2026-04-17-auth-미팅.md`
  - 날짜가 중요하지 않은 맥락: `react-server-components.md`
- 기존 문서와 충돌하면 구분할 수 있도록 조정한다

### 위키링크

적극적으로 위키링크를 생성한다:

- **날짜**: 기록이 해당하는 날짜를 항상 링크한다 (`[[2026-04-17]]`)
- **사람**: 언급된 모든 사람을 링크한다 (`[[김철수]]`)
- **주제**: 기존 document와 매칭되면 링크한다. 기존 document가 없더라도 독립 document로 존재할 가치가 있는 주제는 링크한다

위키링크는 본문 내에 자연스럽게 배치한다. 별도 섹션으로 몰아넣지 않는다.

### 본문

- 프론트매터 없음
- 형식 제약 없음
- 모든 문서는 ~/.claude/shulker/ 루트에 플랫하게 위치한다

### 참조 document

위키링크 대상 중 아직 존재하지 않는 document는 stub을 생성한다.

- 사람: 이름을 제목으로 한 빈 문서
- 날짜: 날짜를 제목으로 한 빈 문서
- 주제: 한 줄 설명을 가진 문서

## 제약

- 목적을 미리 정의된 모드로 제한하지 않는다 -- 사용자의 자유 목적을 받는다
- document 접근은 aeira search/graph를 통한다
- 기존 document를 수정할 때는 원래 내용을 존중하고, 위키링크 추가/보완에 한정한다
