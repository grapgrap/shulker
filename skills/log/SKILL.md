# Log

사용자의 맥락을 ~/.claude/shulker/에 document로 기록한다.

## 흐름

### 1. 입력 파악

사용자의 발화에서 기록할 내용을 파악한다.

- 내용이 충분하면 그대로 기록한다
- 맥락만 있고 내용이 부족하면 무엇을 기록할지 물어본다

### 2. 기존 document 검색

입력에서 핵심 엔티티(사람, 주제, 날짜)를 추출하고, 각각에 대해 기존 document를 검색한다.

```bash
aeira search -s ~/.claude/shulker/ "{엔티티}"
```

매칭되는 기존 document가 있으면 위키링크 대상으로 사용한다.

### 3. 문서 작성

작성 규칙을 따라 document를 생성한다.

- 사용자가 전달한 내용의 맥락과 구조를 살려 작성한다

### 4. 참조 document 생성

작성 규칙의 참조 document 항목을 따른다.

### 5. 동기화

```bash
aeira sync -s ~/.claude/shulker/
```

## 작성 규칙

document를 생성하거나 수정할 때 따르는 규칙이다.

### 파일 이름

- 사람: `김철수.md`
- 날짜: `2026-04-17.md`
- 그 외: 최대한 짧은 단어로 구성한다
  - 날짜가 중요한 맥락: 날짜를 앞에 둔다. `2026-04-17-auth-미팅.md`
  - 날짜가 중요하지 않은 맥락: `react-server-components.md`
- 기존 문서와 충돌하면 구분할 수 있도록 조정한다

### 위키링크

적극적으로 위키링크를 생성한다:

- **날짜**: 기록이 해당하는 날짜를 항상 링크한다 (`[[2026-04-17]]`)
- **사람**: 언급된 모든 사람을 링크한다 (`[[김철수]]`)
- **주제**: 기존 document와 매칭되면 링크한다. 기존 document가 없더라도 독립 document로 존재할 가치가 있는 주제는 링크한다

위키링크는 본문 내에 자연스럽게 배치한다. 별도 섹션으로 몰아넣지 않는다.

### 본문

- 프론트매터 없음
- 형식 제약 없음
- 모든 문서는 ~/.claude/shulker/ 루트에 플랫하게 위치한다

### 참조 document

위키링크 대상 중 아직 존재하지 않는 document는 stub을 생성한다.

- 사람: 이름을 제목으로 한 빈 문서
- 날짜: 날짜를 제목으로 한 빈 문서
- 주제: 한 줄 설명을 가진 문서

## 제약

- 위키링크가 유일한 관계 표현 수단이다
